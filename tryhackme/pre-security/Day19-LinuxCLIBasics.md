📚 What I Learned

The Linux CLI allows direct interaction with the OS through typed commands in a terminal

The file system follows a hierarchical structure starting from the root directory /
Permissions are assigned to owner, group, and others — controlling who can read, write, or execute files

Piping (|) and redirection (>, >>) allow chaining commands and saving output to files

The CLI is the primary interface for managing Linux servers — essential for IT support and cybersecurity roles

💻 Commands Explored

Command	What it does
pwd	Prints the current working directory
ls	Lists files and directories in the current location
ls -la	Lists all files including hidden ones with detailed permissions
cd	Changes the current directory
mkdir	Creates a new directory
touch	Creates a new empty file
cp	Copies a file or directory to a new location
mv	Moves or renames a file or directory
rm	Removes a file (use -r for directories)
cat	Displays the full contents of a file
less	Views file content one page at a time
grep	Searches for a pattern or string within files
find	Searches for files and directories by name or type
chmod	Changes file or directory permissions
chown	Changes the owner of a file or directory
sudo	Runs a command with superuser (admin) privileges
whoami	Displays the currently logged-in username
echo	Prints text or variable values to the terminal
man	Opens the manual/help page for a command
history	Shows a list of previously run commands


🔐 Why It Matters

Linux powers the majority of web servers and cloud infrastructure — CLI fluency is non-negotiable in cybersecurity

Most penetration testing and forensics tools run on Linux — knowing the CLI means working faster and more effectively
