📚 WHAT I LEARNED

The client-server model is the foundation of most networked applications — clients request resources, servers provide them

Servers can host multiple services simultaneously, such as web, email, and file sharing, each on different ports

The request-response cycle underpins how browsers, apps, and APIs communicate with back-end systems

Authentication and authorisation control which clients can access which server resources
Vulnerabilities in either the client or server can be exploited — both sides of the model need to be secured
