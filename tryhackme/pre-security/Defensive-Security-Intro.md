📚 What I Learned (3–5 lines max)
Defensive security is all about monitoring systems and activities performed on them to provide optimal protection against attacks or stop an ongoing attack by following various security practices and policies to contain and curtail these attacks. 
🛠️ What I Did
In the lab for this section, I looked through a simulated Intrusion Detection System (IDS) dashboard to discover an abnormality that has occurred, I identified it was an automated web discover attack as identified by the IPS tool, I took note of the source IP address and finally blocked the IP address through a simple firewall block setting.


🔐 Why It Matters
I believe this lesson matters because Defensive security is the engine upon which systems depend on to maintain the security triage of Confidentiality, integrity and Authentication (CIA). 
Attention to details mattered, which helped discover the abnormality, source of attack and ultimately helped decide the containment action taken.

❓ One Thing I Didn’t Fully Understand
What I am curious about is how to stop this attack from repeating. From the resolution, I was able to block the IP address of the attacker, they can attack again using a different IP address which hasn’t been blocked yet. I will try to understand how to completely prevent the Web discovery attack from even discovering any vulnerable URL in the first place.
