📚 WHAT I LEARNED
Plaintext — The original, readable message before encryption.

Ciphertext — The encrypted, scrambled message after applying the cipher.

Key — A secret value that controls how the cipher transforms the message.

Caesar Cipher — A substitution cipher that shifts each letter by a fixed number of positions.

Brute Force — Trying all possible keys until finding the correct one.


🛠️ WHAT I DID
Used the Caesar Cipher to encrypt and decrypt messages, just to get a feel for how it works — while noting it is not to be used in real world security because it's so basic and can be cracked in milliseconds.

Explored how brute force works against the Caesar Cipher by cycling through all 25 possible shift values to recover the original plaintext.

Examined how symmetric encryption uses a single shared key for both encrypting and decrypting a message, contrasting it with the simplicity of classical ciphers.


🔐 WHY IT MATTERS
Understanding basic ciphers like Caesar reveals the core principles behind all encryption — confidentiality, keys, and the cost of breaking them.

Brute force attacks are still used today against weak encryption — knowing why simple ciphers fail helps you appreciate why modern algorithms like AES exist.
