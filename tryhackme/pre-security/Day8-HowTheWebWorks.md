📚 WHAT I LEARNED

HTTP is the protocol browsers use to request and receive web content from servers

HTTP methods define the action being requested — GET retrieves data, POST submits it, PUT updates, DELETE removes

Status codes communicate the result of a request — 200 is success, 301 is redirect, 404 is not found, 500 is server error

HTTPS is the encrypted version of HTTP using TLS, protecting data in transit from interception

HTTP headers carry metadata about requests and responses — including cookies, content type, and caching rules

🔐 WHY IT MATTERS

Most web attacks — like SQL injection, XSS, and CSRF — are delivered over HTTP, so understanding the protocol is foundational

Knowing how headers and cookies work helps identify session hijacking and man-in-the-middle attack vectors
